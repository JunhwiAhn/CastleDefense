### MinimalCharacters / SPUM について
- MinimalCharactersのキャラクター(character_1〜8)はボディパーツ分解型(Head, Body, Leg, Weapon等の個別PNG)
- SPUMも同様にパーツ分解型のスプライトシステム
- Flameで使用するには各パーツを組み合わせてCompositeスプライトを構築するか、事前にPSD/PSBから合成画像を書き出す必要あり
- 現行の単一画像描画方式では直接使用困難

## 6. castle_defense_game.dart での読み込み提案

### Flame での画像読み込み
```dart
// onLoad() 内で画像をプリロード
@override
Future<void> onLoad() async {
  // モンスター画像
  await images.load('monsters/monster_stage1_slime.png');
  await images.load('monsters/miniboss_cerberus_fire.png');

  // UI画像
  await images.load('ui/btn_red.png');
  await images.load('ui/panel_dark_purple.png');
  await images.load('ui/bar_background.png');
  await images.load('ui/bar_red.png');

  // アイコン画像
  await images.load('icons/icon_buff_attack.png');
  await images.load('icons/icon_element_fire.png');
  await images.load('icons/icon_hud_heart.png');
}
```

### SpriteComponent での使用例
```dart
// モンスタースプライト
final monsterSprite = SpriteComponent(
  sprite: Sprite(images.fromCache('monsters/monster_stage1_slime.png')),
  size: Vector2(40, 40), // ゲーム内表示サイズ
);

// HPバー (NineTileBox推奨)
final hpBarBg = SpriteComponent(
  sprite: Sprite(images.fromCache('ui/bar_background.png')),
  size: Vector2(100, 10),
);
```

### Canvas での使用例 (現行方式)
```dart
// render() 内で直接描画
void _renderMonster(Canvas canvas, _Monster m) {
  final img = images.fromCache('monsters/${m.spriteKey}.png');
  canvas.drawImageRect(
    img,
    Rect.fromLTWH(0, 0, img.width.toDouble(), img.height.toDouble()),
    Rect.fromCenter(center: Offset(m.x, m.y), width: 40, height: 40),
    Paint(),
  );
}
```

## 7. リサイズが必要な素材

| ファイル | 現在サイズ | 推奨サイズ | 理由 |
|---|---|---|---|
| `ui/panel_dark_*.png` | 1212x1115 | 390x360 or NinePatch | メモリ節約、基準解像度390x844に合わせる |
| `icons/icon_element_*.png` | 512x512 | 64x64 or 128x128 | HUDアイコンとして過大 |
| `icons/icon_buff_*.png` | 512x512 | 64x64 or 128x128 | バフカードアイコンとして過大 |
| `icons/icon_skill_bomb.png` | 512x512 | 128x128 | スキルボタン用 |
| `ui/menu_background.png` | 2560x1600 | 390x844 | 基準解像度に合わせる |

> Flameの`Sprite`描画では表示サイズを指定できるため、即座のリサイズは必須ではないが、
> メモリ効率を考慮するとリサイズ推奨。特に`menu_background.png`(2560x1600)は
> モバイルで不必要に大きい。
