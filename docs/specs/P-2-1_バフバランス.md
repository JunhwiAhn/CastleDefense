## バフの出現抽選ロジック推奨

### 出現重み
- 基本全バフが等確率（各12.5%）
- 調整: **直前の選択と同じバフの出現確率を50%削減**（連続同じバフの防止）

### 除外条件（その選択肢に出さない）
- 最大重複回数に達したバフは除外
- 残り選択肢が3種未満になった場合は除外条件を緩和（「城修復」「城バリア」は必ず補填）

### バランス例（理想的な3択構成）
- 1択: メインキャラ強化系（攻撃力/速度/移動速度/射程のいずれか）
- 1択: タワー/城系（タワー強化/城修復/城バリアのいずれか）
- 1択: ユーティリティ（XP磁石/その他）

> この構成はあくまで「参考の傾向」として抽選の後処理で調整する。完全ランダムより選択に深みが出る。

## Engineer向け実装参考

```dart
// バフ適用の多重スタック計算
// 攻撃力UP: multiplicative (乗算)
mainCharAtk = baseAtk * pow(1.15, atkUpCount);

// 攻撃速度UP: multiplicative (乗算) ← 間隔を減らす
mainCharAtkInterval = baseInterval * pow(0.90, spdUpCount);

// 移動速度UP: multiplicative
mainCharMoveSpeed = baseSpeed * pow(1.20, moveUpCount);

// 射程UP: multiplicative
mainCharRange = baseRange * pow(1.15, rangeUpCount);

// タワー強化: multiplicative (全タワーに適用)
for (tower in towers) {
  tower.atk = tower.baseAtk * pow(1.10, towerUpCount);
}

// XP磁石拡大: additive (加算)
xpMagnetRadius = 20 + (15 * magnetCount); // 20 + (15 × n)
```
