# 増強(オーグメント)システム仕様書

> 作成日: 2026-03-11
> 参照: docs/research/augment-benchmark.md / docs/specs/element-system.md

---

## 概要

増強(オーグメント)システムは、ゲームの各ステージを通じて**ビルドを決定づける強力な恒久強化**を取得するシステムです。
レベルアップ時のバフカード（頻繁・小強化）とは明確に異なり、**少頻度・高インパクト**の選択を提供します。

### バフカードとの差別化

| 項目 | レベルアップバフカード | 増強(オーグメント) |
|------|---------------------|-----------------|
| 発生頻度 | レベルアップ毎（高頻度） | ラウンド節目3回/ステージ |
| 効果規模 | 小〜中（累積で強化） | 中〜大（即座にビルド方向を決定） |
| 選択数 | 3択 | 3択 |
| 視覚演出 | シンプルカード | 豪華演出・レアリティ色付き |
| タイミング | ゲーム一時停止 | ラウンドクリア画面内で選択 |
| リセット | ステージ持越しなし | ステージ間持越し（後述） |

---

## 選択タイミング

ステージ内で**合計3回**の増強選択機会が発生します。

| タイミング | 発生条件 | 提示ティア確率 |
|-----------|---------|-------------|
| **増強 I** | ラウンド2クリア後 | Common60% / Rare35% / Legendary5% |
| **増強 II** | ラウンド4クリア後 | Common25% / Rare50% / Legendary25% |
| **増強 III** | ラウンド5(ボス)クリア後 | Common10% / Rare40% / Legendary50% |

- 各タイミングで**3択**から1つを選択（選択しない2つは消える）
- 選択はラウンドクリア画面（既存の3秒インターバル画面）に続いて表示
- 選択中はゲームが一時停止
- タイムアウトなし（モバイル考慮）

---

## レアリティ(ティア)システム

### 3段階ティア

| ティア | 表示色 | カード枠色 | 強さの目安 |
|--------|--------|----------|---------|
| **一般 (Common)** | 白/グレー | `#9E9E9E` | 汎用的な補助効果 |
| **希少 (Rare)** | 青 | `#2196F3` | 顕著なDPS/耐久向上 |
| **伝説 (Legendary)** | 金〜虹 | `#FFD700` | ビルドを定義するレベルの強力効果 |

### 同一増強の重複防止
- 同じラン内で同一の増強は二度提示されない
- 選択済みの増強と効果カテゴリが完全に被る場合、低確率でのみ再提示

## UI・プレゼンテーション

### 選択画面レイアウト

```
┌─────────────────────────────┐
│     ⚡ 増強選択 ⚡           │  ← タイトル(ティア発表演出付き)
│                             │
│  ┌──────┐ ┌──────┐ ┌──────┐ │
│  │ [一般] │ │[希少]│ │[伝説] │ │  ← ティア色のカード枠
│  │ アイコン│ │アイコン│ │アイコン│ │
│  │        │ │      │ │      │ │
│  │ 増強名 │ │ 増強名 │ │ 増強名│ │
│  │        │ │      │ │      │ │
│  │ 効果説明│ │効果説明│ │効果説明│ │
│  └──────┘ └──────┘ └──────┘ │
└─────────────────────────────┘
```

### カード演出
- ラウンドクリア画面（「ROUND CLEAR!」→獲得XP/ゴールド表示）の後にフェードイン
- カード出現時: 各カードが上からスライドイン（0.3秒、0.1秒ずつずれ）
- 伝説カード選択時: 金色の光エフェクト + 短いシェイク演出
- 選択後: 未選択カードがフェードアウト、選択カードが中央に拡大表示（0.5秒）
- 再開: 選択確定後に次ラウンド開始

### ステージクリア後の確認
- ステージクリア（ゴールドショップ）画面に「取得済み増強」一覧を表示
- 次ステージ持越しの増強は明示的にアイコン表示

## ステージ間の持越しルール

| 設定 | 内容 |
|------|------|
| **基本**: ステージ内有効 | 取得した増強はそのステージのみ有効 |
| **例外**: 伝説増強の一部 | Legendary増強は次ステージにも持越し（該当増強に明記） |
| **リセット**: ステージクリア後 | 一般・希少増強はリセット、伝説増強のみ持越し |

> ※ 持越しルールは初期仕様。テストプレイ結果により「全ステージ持越し」に変更する可能性あり。

## 属性システムとの連携

属性システム（`docs/specs/element-system.md`）との統合:

| 増強名 | 連携内容 |
|--------|---------|
| C-14「属性目覚め」 | 属性ボーナス倍率 x1.5→x1.6へ向上（属性マスタリーと同効果） |
| R-07「元素爆発」 | 通常攻撃に属性状態異常発動確率を付与（火傷/凍結/束縛/感電/呪い） |
| L-05「元素の嵐」 | チーム内3属性以上でランダム属性状態異常を全攻撃に付与（30%確率） |

### 属性シナジーボーナス（増強なしでも発動）
| チーム内属性数 | ボーナス |
|-------------|---------|
| 2属性 | なし |
| 3属性 | 全キャラのダメージ+5% |
| 4属性以上 | 全キャラのダメージ+10%、属性状態異常時間+0.5秒 |

## 既存バフカードとの整合性

### 重複・上乗せルール
- 増強とバフカードの効果は**独立して乗算**される
  - 例: バフカード「攻撃速度UP×3」(×0.729) + 増強「連射の才」(×1.2) = 0.729×1.2 = 0.875の攻撃間隔
- 増強L-08「永遠の契約」のみバフカードスタック数に干渉する（特殊ルール）

### バフカード選択肢への影響
- 増強「超集中砲火」(R-05) は既存の「タワー強化」バフと別軸の強化であり競合しない
- 増強「鉄壁の城」(R-08) と「城バリア」バフカードは異なる効果（恒常%削減 vs 一時無効）

## 実装タスク（エンジニア向け参考）

### データ構造案
```dart
enum AugmentTier { common, rare, legendary }
enum AugmentCategory { main, tower, castle, utility, economy, elemental, special, synergy }

class Augment {
  final String id;        // 例: 'C-01'
  final String nameJp;    // 表示名
  final AugmentTier tier;
  final AugmentCategory category;
  final String description;
  final bool carryOverToNextStage; // ステージ間持越しフラグ
  // 効果はapply(GameState)メソッドで実装
}
```

### 選択ロジック
```dart
// ラウンドクリア時に呼ばれる増強選択開始処理
void _triggerAugmentSelection(int round) {
  if (round != 2 && round != 4 && round != 5) return;

  // ティア確率テーブル
  final tierProbs = {
    2: {AugmentTier.common: 0.60, AugmentTier.rare: 0.35, AugmentTier.legendary: 0.05},
    4: {AugmentTier.common: 0.25, AugmentTier.rare: 0.50, AugmentTier.legendary: 0.25},
    5: {AugmentTier.common: 0.10, AugmentTier.rare: 0.40, AugmentTier.legendary: 0.50},
  };

  // 3択を生成（既取得・同一ID除外）
  final choices = _generateAugmentChoices(tierProbs[round]!);
  _showAugmentSelectionUI(choices);
}
```

### 必要な新規実装
- `AugmentData`クラス / 全34増強の定義
- `_showAugmentSelectionUI()` — カード表示・選択UI
- `_applyAugment(Augment)` — 各増強の効果適用
- ゲーム状態に`List<Augment> activeAugments`フィールド追加
- ラウンド2/4/5クリア時のトリガー処理

## バランス考慮事項

1. **Legendary増強の頻度管理**: ステージ1では伝説確率5%/25%/50%と設定。後半ステージで同確率を維持するか、または「取得済み伝説数が多い場合は確率を下げる」調整も検討。

2. **伝説L-01(不死の誓い)の判定**: HP=0を瞬間的に検知する必要あり。`_handleMainCharDeath()` フックに割込み処理を追加する形で実装推奨。

3. **タワー増強(R-03爆発弾頭)の適用範囲**: メインキャラのみか全タワーにも適用するか要テスト。全タワー適用では後半のDPSが大幅に上がりすぎる可能性あり。

4. **L-08(永遠の契約)のバフカード上限超過問題**: バフカードの最大重複上限（例: 攻撃力UP=5回）を超えさせる「+1スタック」は、上限を変えずに効果のみ1回分追加適用する方式で実装。
