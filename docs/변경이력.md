## v0.5 — 투사물 비주얼 개선 / VFX 이펙트 시스템 추가 (Designer)

> 담당: **디자이너** | Hyper Casual 스타일 Canvas 커스텀 렌더링

### 1. 역할별 투사물 비주얼 분리

기존에는 모든 투사물이 "작은 원 + 꼬리 선"으로 동일했으나, 역할별로 구분되는 비주얼로 개선.

| 역할 | 비주얼 | 구현 방식 |
|------|--------|-----------|
| 궁수 (Archer) | ◁━━ 삼각형 화살촉 + 몸통 선 | `drawPath()` 삼각형 + `canvas.rotate()` 진행방향 회전 |
| 총잡이 (Gunslinger) | ○○○● 잔상 트레일 + 탄환 | `trail[]` 3프레임 위치 기록, alpha 감쇠 |
| 마법사 (Mage) | ✦ 펄스하는 광구 + 회전 파티클 | `sin(gameTime*10)` 크기 펄스, 2개 궤도 파티클 |
| 성직자 (Priest) | + 십자형 | `drawLine()` 십자, 연두색 |

#### 코드 변경 (`castle_defense_game.dart`)

**_Projectile 클래스 확장**
```dart
ClassType? sourceClass;  // 궁수/총잡이 구분용 (둘 다 physicalDealer이므로)
List<Vector2> trail;     // 잔상 위치 기록 (최대 3개, 총잡이용)
```

**`_renderProjectiles()` 전면 개편**
- 기존: `sourceRole` 기반 색상 분기 → 원 + 꼬리
- 개선: `sourceClass` / `sourceRole` 기반 형상 분기 → 역할별 고유 비주얼
- `_fireProjectile()`, `_handleMagicUnit()`, 궁수 3발 발사 등 모든 투사물 생성부에 `sourceClass` 전달 추가

### 2. VFX 이펙트 시스템 신규 추가

범용 이펙트 관리 시스템을 도입하여 4종의 VFX를 구현.

**데이터 구조**
```dart
enum VfxType { hit, death, shockwave, barrier }

class _VfxEffect {
  Vector2 pos;
  double timer, duration;
  VfxType type;
  double maxRadius;
  Color color;
  // progress, isExpired 등 유틸리티 getter
}
```

**라이프사이클**: `_updateVfxEffects(dt)` → timer 누적 → 만료 시 자동 제거

| VFX | 트리거 | 연출 | 지속 |
|-----|--------|------|------|
| 히트 스파크 | `_damageMonster()` | 5개 방사형 흰색 선 확산 + 소멸 | 0.2초 |
| 죽음 파프 | 몬스터 HP ≤ 0 | Phase1: 원 팽창 → Phase2: 6파티클 비산 | 0.4초 |
| 충격파 | 필살기 발동 시 (**렌더링만 구현**, 트리거는 Engineer #59 대기) | 확대 링 + 중앙 플래시 | 0.5초 |
| 배리어 | 버프 선택 시 (**렌더링만 구현**, 트리거는 Engineer #65 대기) | 회전 6각형, 잔여 3초 점멸 | 10초 |

- 죽음 파프 색상: 보스=빨강, 미니보스=주황, 일반=노랑

#### 렌더링 파이프라인 변경
```
render() 호출 순서:
  ... → _renderProjectiles() → _renderVfxEffects() ← 신규 → _renderMonsters() → ...
```

## v0.4 — 공격 방식 개선 / 사거리 조정 / Goblin 스프라이트 적용

### 1. 클래스별 공격 방식 구현

| 클래스 | 공격 방식 | 상세 |
|--------|-----------|------|
| 마법사 (Magic Dealer) | 스플래시 데미지 | 투사물이 적에게 충돌하면 반경 50px 내 모든 적에게 데미지. 보라색 빛나는 구체로 시각화됨 |
| 궁수 (Archer) | 3발 동시 발사 | 부채꼴 모양으로 3발 동시 발사. 중앙 화살은 유도, 좌우 화살은 약 15도(0.25rad) 간격 직선 발사 |
| 총잡이 (Gunslinger) | 5발 연속 발사 | 공격 시 0.08초 간격으로 5발 연속 발사 (버스트 샷) |

- 관련 함수: `_handleMagicUnit`, `_handleArcherUnit`, `_handleGunslingerUnit`

---

### 2. 원거리 유닛 사거리 조정

#### 1차 조정 (물리 딜러 / 힐러 증가)
| 유닛 타입 | 변경 전 | 변경 후 |
|-----------|---------|---------|
| 물리 딜러 (궁수, 총잡이 등) | 250px | 350px |
| 힐러 (성직자) | 250px | 400px |

#### 2차 조정 (전체 원거리 유닛 1.5배 증가)
| 유닛 타입 | 변경 전 | 변경 후 |
|-----------|---------|---------|
| 기본 원거리 (`rangedRange`) | 250px | 375px |
| 물리 딜러 (`physicalDealerRange`) | 350px | 525px |
| 힐러 (`priestRange`) | 400px | 600px |

- 관련 상수: `rangedRange`, `physicalDealerRange`, `priestRange`

### 3. 캐릭터 상단 이동 제한

- 상단 100px(`topBoundary`) 영역에 캐릭터가 올라가지 못하도록 제한
- 게임 UI(몬스터 카운터, 타이머, 스테이지 표시) 영역과 캐릭터가 겹치는 문제 해결
- 모든 유닛 타입(근거리, 원거리, 궁수, 총잡이, 마법사)의 이동 로직에 `unit.pos.y.clamp(topBoundary, size.y)` 적용

### 4. Goblin 스프라이트 적용 (Stage 1 일반 몬스터)

#### 추가 파일
- `assets/images/goblin.png` — `goblin_sample2.png` 기반의 도트 아트 스프라이트

#### pubspec.yaml 변경
```yaml

    - assets/images/
```

#### 코드 변경 (`castle_defense_game.dart`)

**_Monster 클래스 — 애니메이션 필드 추가**
```dart
double animationTimer = 0.0; // 애니메이션 타이머
int currentFrame = 0;        // 현재 프레임 (0~3)
```

**애니메이션 업데이트**
- 매 프레임 `animationTimer`를 누적, 0.15초마다 `currentFrame`을 0→1→2→3→0 으로 순환
- 걷기(`walking`) 및 낙하(`falling`) 중에만 갱신

**렌더링**
- Stage 1 일반 몬스터에 한해 `_renderGoblinSprite()` 호출
- 보스 / 미니보스는 기존 원형 렌더링 유지
- `currentFrame` 값에 따라 `[0.0, -2.0, 0.0, 2.0]` px 상하 바운스 적용
- 데미지 플래시 시 `ColorFilter.mode(Color(0xFFFF0000), BlendMode.srcATop)` 적용
