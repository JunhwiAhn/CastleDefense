# タスク分担表 — チーム別作業リスト

> 新ゲーム仕様書.md の実装に必要な全タスクをチーム別に分類

---

## チーム構成

| 役割 | 担当領域 |
|------|---------|
| **기획자 (Planner)** | 仕様策定、バランス調整、他ゲームベンチマーク、実装評価・レビュー |
| **디자이너 (Designer)** | キャラデザイン、UI/UXレイアウト設計、画面演出実装 |
| **백엔드 엔지니어 (Backend)** | ゲームロジック、システム実装、データ構造、最適化 |

## 기획자 (Planner) タスク一覧

### P-1. ベンチマーク調査

| # | タスク | 詳細 | 参考ゲーム例 |
|---|--------|------|-------------|
| P-1-1 | レベルアップバフ設計のベンチマーク | Vampire Survivors風ゲームのレベルアップ選択肢を調査。バフの種類、バランス、出現確率の参考値を収集 | Vampire Survivors, Brotato, Halls of Torment |
| P-1-2 | タワーディフェンスのバランス調査 | 城HP・タワー射程・敵HP/速度のバランス感覚を他ゲームから収集 | Kingdom Rush, Arknights |
| P-1-3 | モバイルスティック操作のUX調査 | バーチャルスティックの操作感（デッドゾーン、感度）のベストプラクティス調査 | Archero, Survivor.io, Brawl Stars |
| P-1-4 | ゴールドショップ経済設計の参考調査 | ステージ間ショップの価格バランス、インフレ曲線を他ゲームから調査 | Vampire Survivors, Hades |
| P-1-5 | スポーンパターン調査 | 面白い敵出現パターン（ラッシュ、包囲、一方向集中）の事例収集 | Vampire Survivors, 20 Minutes Till Dawn |

### P-2. 仕様策定・詳細化

| # | タスク | 詳細 |
|---|--------|------|
| P-2-1 | バフカードのバランス数値確定 | 8種バフの効果量・最大重複回数の最終値を決定。P-1-1の調査結果を反映 |
| P-2-2 | ラウンド進行テーブルの精緻化 | ラウンド6以降の敵構成（種類混合比率、ミニボス出現頻度）を定義 |
| P-2-3 | 敵ステータス表の作成 | モンスター種別ごとのHP/攻撃力/速度/ドロップ率を一覧表にまとめる |
| P-2-4 | ステージ間ショップ品目・価格表の確定 | ゴールド経済が破綻しないよう、収入と支出のバランスシートを作成 |
| P-2-5 | 役割別キャラのメイン/タワー適性ガイド | 各役割(궁수,총잡이,마법사等)がメイン操作とタワー配置でどう違うか整理 |
| P-2-6 | 難易度カーブ設計書 | ステージ1〜10の想定プレイ時間、レベル到達目安、城HP推移シミュレーション |

### P-3. 評価・レビュー

| # | タスク | 詳細 |
|---|--------|------|
| P-3-1 | プレイテスト評価シート作成 | テスト項目チェックリスト（操作感、難易度、テンポ、爽快感）を準備 |
| P-3-2 | 実装後バランスレビュー | 実装された各システムをプレイして数値バランスの問題点を報告 |
| P-3-3 | UXフロー評価 | ゲーム開始→プレイ→レベルアップ→ラウンドクリア→ショップの一連の流れを評価 |
| P-3-4 | バフカード体感テスト | 各バフが体感で分かるか、選択に悩む面白さがあるかを評価 |

## 実装順序（推奨フェーズ）

```
Phase 1: コア基盤 ─────────────────────────────────
  Backend : B-1-1〜B-1-8 (城中央化、スポーン、移動、fallingの削除)
  Designer: D-1-3 (バーチャルスティック), D-4-1 (城スプライト), D-4-2 (タワー枠)
  Planner : P-1-1〜P-1-5 (ベンチマーク調査)

Phase 2: 操作 & 戦闘 ──────────────────────────────
  Backend : B-2-4〜B-2-7 (スティック移動、メインキャラHP、復活)
  Backend : B-2-1〜B-2-3 (城HP、ダメージ、聖職者回復)
  Designer: D-1-1 (HUD), D-1-2 (城HPバー), D-2-4 (復活カウントダウン)
  Designer: D-3-1 (城ダメージ点滅), D-3-6 (復活無敵演出)
  Planner : P-2-3 (敵ステータス表)

Phase 3: 成長システム ──────────────────────────────
  Backend : B-2-8〜B-2-12 (XPジェム、レベルアップ、バフ、マグネット)
  Designer: D-3-2 (XPジェム描画), D-3-4 (マグネット演出)
  Designer: D-2-1 (バフ選択UI), D-4-5 (バフカードアート)
  Planner : P-2-1 (バフバランス確定), P-2-6 (難易度カーブ)

Phase 4: 必殺技 & ゴールド ─────────────────────────
  Backend : B-2-13〜B-2-16 (必殺技、ゴールド、ショップ)
  Designer: D-1-4 (SKILLボタン), D-3-5 (ボムエフェクト)
  Designer: D-1-5 (ゴールド表示), D-3-3 (コイン描画), D-2-2 (ショップUI)
  Planner : P-2-4 (ショップ経済バランス)

Phase 5: ラウンド進行 & 仕上げ ─────────────────────
  Backend : B-2-17〜B-2-20 (ラウンド進行、インターバル、上限、バリア)
  Backend : B-3-1〜B-3-3 (スケーリング、最適化)
  Designer: D-1-6 (ラウンドクリア演出), D-2-3 (ゲームオーバー)
  Designer: D-3-7 (バリアエフェクト), D-4-3 (ボス), D-4-4 (ミニボス)
  Planner : P-2-2 (ラウンドテーブル精緻化), P-2-5 (役割適性ガイド)

Phase 6: テスト & バランス調整 ─────────────────────
  Planner : P-3-1〜P-3-4 (全評価タスク)
  全員    : フィードバック反映、数値調整、バグ修正
```
