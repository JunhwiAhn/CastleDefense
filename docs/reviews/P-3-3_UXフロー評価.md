# P-3-3 UXフロー評価

> GitHub Issue #14 対応
> 担当: Planner
> 評価日: 2026-03-11
> 対象ビルド: コミット 60905d1 時点

---

## 1. 概要

ゲーム全体のUXフローを「ゲーム開始 → プレイ → レベルアップ → ラウンドクリア → 増強選択 → ショップ → ゲームオーバー」の一連の流れで評価した。

**総合所感**: 基本的なゲームフローは仕様通りに実装されており、GameState遷移は論理的に正しい。Violet Themeを軸としたUIデザインの統一感もある。一方で、情報の重複表示、言語の混在、スティック入力の改善余地、初見プレイヤー向けガイダンスの不足など、リリース前に対応すべき課題がいくつか存在する。

**推定UXフロースコア: 7/10** (合格ラインの8/10にはあと一歩)

## 2. ゲームフロー分析

### GameState遷移パス

```
loading (0.5秒) → roundSelect → playing
                                  ↓
                            [城HP=0] → result (GameOver)
                            [時間切れ] → result (GameOver)
                            [ラウンドクリア] → roundClear (3秒)
                                              ↓
                                        [R2/R4/R5] → augmentSelect → playing (次ラウンド)
                                        [その他] → playing (次ラウンド)
                            [全ラウンドクリア] → shopOpen → result (クリア)
                            [レベルアップ] → levelUp → playing
                            [一時停止] → paused → playing / roundSelect / playing(リトライ)
```

### 評価

**良い点:**
- 状態遷移が明確で、各GameStateに対応するupdate/render/tapHandlerが分離されている
- ラウンドクリア→増強選択→次ラウンドの流れが自然
- 一時停止中は全ゲームロジックが停止し、不意のダメージが発生しない
- レベルアップ中もXPマグネットタイマーだけ進行する設計は、ゲームテンポの維持に良い

**問題点:**
- **P-01: ローディングが0.5秒で形式的** — ローディング画面は「준비 중...」(韓国語)のみ表示。ロード完了後はroundSelectに即遷移するが、実際のアセット読み込み(5画像の非同期ロード)はonLoadで完了しているため、この0.5秒は無意味な待機時間
- **P-02: roundSelect→playingの遷移が即座** — ラウンドノードをタップすると即座にplaying状態になる。戦闘開始前のカウントダウン(3, 2, 1, GO!)がなく、スティック操作の準備ができないまま敵がスポーンし始める
- **P-03: 時間切れとゲームオーバーの区別がない** — `_onTimeOver()`と`_onGameOver()`は同じresult画面に遷移するが、プレイヤーには「城HP=0で負けた」のか「時間切れで負けた」のか区別がつかない

## 3. HUD・情報表示

### 現状のHUDレイアウト (390x844基準)

```
┌─────────────────────────────────────┐ y=0
│ [HP■■■░░] Lv.1  [XP■■░░░ Lv.1 XP] │ Row1 (y=14): HPバー+レベル+XPバー
│ 👾12   ⏱ 1:30   Stage 1-2          │ Row2 (y=32): モンスター数+タイマー+ステージ
│ 🪙 30G                          🔥  │ Row3 (y=57): ゴールド+属性アイコン
├─────────────────────────────────────┤ y=75
│                                     │
│ (StageProgress重複: 👾12  0:30  1-2)│ y=15: marginTop=15
│ (WeaponInfo: 기본검 DMG:1)          │ y≒(castleHeight付近)
│ (PauseButton: ⏸)                   │ 右上
│         [Castle+TowerSlots]         │ 画面中央
│                                     │
│ 🕹️               🪙30G  [💥SKILL]  │ 左下/右下
└─────────────────────────────────────┘ y=844
```

### 評価

**良い点:**
- HUD背景が半透明黒(`0xCC000000`)で戦闘画面に干渉しにくい
- HPバーの色が緑→黄→赤のグラデーションで直感的
- 属性アイコンがHPバー横と右下の2箇所に表示され、属性状態が把握しやすい
- ゴールド表示がHUD内と右下の両方にあり、常に確認可能

**問題点:**
- **P-04: 情報の重複表示が深刻** — `_renderStageProgress()`と`_renderHUD()`の両方がモンスター数・タイマー・ステージ番号を描画しており、画面上部に同じ情報が2段で表示される。`_renderStageProgress()`はy=15付近に大きいフォント(fontSize=20-24)で、`_renderHUD()`はy=32付近にfontSize=12で描画。どちらが正式な情報源か混乱する
- **P-05: WeaponInfo「기본검 (DMG: 1)」が韓国語のまま** — ゲーム内UIテキストの一部が韓国語で、一貫性がない（HUDの他の要素はLv., Stage, XPなど英語表記）
- **P-06: WeaponInfoの位置が不自然** — `size.y - castleHeight + padding`の位置は画面下部(y≒772)で、スティックやスキルボタンと重なる可能性がある。戦闘中に「基本剣」の情報を常時表示する必要性が疑問
- **P-07: HUDの高さ(60+18=78px)が基準解像度の約9%** — 844pxのうち78pxをHUDが占有し、戦闘領域を圧迫。特にRow3のゴールド表示は右下にも独立表示があるため冗長

## 4. 操作性評価

### バーチャルスティック

**仕様確認:**
- 外径: 60px / ノブ径: 25px / デッドゾーン: 7px
- 固定位置: (80, size.y - 110) = (80, 734)
- 速度計算: アナログ(倒し量に比例)

**良い点:**
- P-1-3調査結果に基づくSurvivor.io型の固定位置が採用されている
- デッドゾーン7px(外径の約12%)はベストプラクティス範囲内
- ノブがクランプされ外径内に収まる実装が正しい
- 十字ガイド線が方向の目安になる

**問題点:**
- **P-08: スティック入力が画面全体で反応する** — `onDragStart`でplaying状態なら位置に関係なく`_stickActive = true`となる。スティック周辺以外の場所(画面右側など)をドラッグしてもスティックが反応するため、スキルボタン付近のスワイプ操作と干渉する可能性がある。P-1-3で推奨されていた「スティック反応範囲: 中心から半径80px」が実装されていない
- **P-09: スティックのビジュアルが入力状態を反映しない** — `_renderVirtualStick`は常に同じ位置にノブを描画する。`_stickKnobPos`のリアルタイム位置がレンダリングに反映されておらず、ノブが動かない静止画になっている。ユーザーはスティック操作中に視覚フィードバックを得られない
- **P-10: スティックのY位置(y=734)とスキルボタン(y=754)が近い** — 垂直方向の距離が20pxしかなく、大きい指ではスティック操作中にスキルボタン領域に触れる可能性がある

### SKILLボタン

**良い点:**
- 円形ゲージの充填率が視覚的に分かりやすい
- 準備完了時の色変化(青→オレンジ)と外光リングで100%到達が明確
- 「READY」テキストとアイコン変化(`⚡`→`💥`)の二重フィードバック

**問題点:**
- **P-11: スキルボタンの判定範囲(半径40px)とビジュアル(半径38px)がほぼ同じ** — タッチターゲットの推奨最小サイズは44x44pt(Apple HIG)。btnRadius=38は若干小さく、急ぎの場面でミスタップが発生しやすい

### 一時停止ボタン

- 右上に配置されているのは標準的で良い
- ただしボタンサイズや位置の定義(`_pauseButtonRect()`)を確認すると、ゲームプレイ中のみ表示されるのは正しい

## 5. 画面遷移・フィードバック

### レベルアップUI (GameState.levelUp)

**良い点:**
- Violet Themeに統一されたカード(暗紫背景+紫ボーダー+紫ヘッダー)
- カードサイズ(100x150px)が3枚を390px幅に適切に収める
- スタック数バッジ(例: 「2/5」)で残り強化回数が一目で分かる
- 「カードをタップして選択」の案内テキストあり

**問題点:**
- **P-12: バフカード説明が英語/韓国語混在** — 基本バフは英語(「ATK +15% (max 5)」)、属性バフは韓国語(「화염 속성 부여 (상덮어쓰기)」)。言語の統一が必要
- **P-13: バフ名が英語略語のみ** — 「ATK UP」「SPD UP」「MOVE UP」はゲーマーには分かるが、カジュアルプレイヤーには不親切。日本語or韓国語の補助表記があると良い
- **P-14: カード選択後のフィードバックがない** — タップ→即`GameState.playing`に戻る。選択確認アニメーションや、バフ適用エフェクト(例: 画面フラッシュ)がないため「本当に選択されたのか」の確認感が薄い

### ラウンドクリア演出 (GameState.roundClear)

**良い点:**
- 3秒のインターバルが仕様通り
- XP/ゴールドのフライインアニメーション(スライドアップ+フェードイン/フェードアウト)が洗練されている
- ボス/ミニボスクリア時の専用テキスト・背景光彩パルスで達成感がある
- 「다음 라운드 준비 중...」の進捗バーで待ち時間の可視化ができている

**問題点:**
- **P-15: ラウンドクリア中にメインキャラの移動ができるが、スティックUIが非表示** — `_renderVirtualStick`はplaying状態でのみ描画される。roundClear中はスティックが消えるが、実際にはキャラクターは`_buffedMoveSpeed`で移動可能(L-04増強時は400%)。スティックなしでどう移動するのか不明瞭
- **P-16: テキスト「다음 라운드 준비 중...」が韓国語** — ラウンドクリアタイトルは英語(「Round X Clear!」)なのにサブテキストが韓国語で統一感がない

### ショップUI (GameState.shopOpen)

**良い点:**
- Violet/Gold Themeが統一されている
- アイテムカードにアイコン+名前+価格+購入回数が集約されており情報密度が適切
- 購入可否(ゴールド足りない/MAX到達)によるカード色・テキスト色の変化が分かりやすい
- 「続ける →」ボタンが明確

**問題点:**
- **P-17: 購入成功/失敗のフィードバックがない** — `_buyShopItem`は内部でbool値を返すが、UIには反映されない。タップしても画面上の変化がゴールド数値の減少のみで、成功/失敗のフラッシュ表示がない
- **P-18: 購入確認ダイアログがない** — P-3-1評価シートで「購入確認が分かりやすい」を要件としているが、タップ即購入で確認ステップがない。誤操作による無駄遣いの恐れがある

### ゲームオーバー画面 (GameState.result, !_lastStageClear)

**良い点:**
- 城崩壊グラフィック(パーツ回転+均裂線)の演出が凝っている
- 赤いビネット効果で敗北の雰囲気が出ている
- 統計情報(ラウンド+討伐数)が確認できる
- 「もう一度」ボタンのレッドテーマが敗北画面に合っている

**問題点:**
- **P-19: 獲得ゴールドが表示されない** — P-3-1評価シートでは「リザルト（到達ラウンド/倒した敵数/獲得ゴールド）が確認できる」を要件としているが、ゴールド情報が欠落している
- **P-20: 時間切れゲームオーバーの専用表示がない** — P-03と関連。`_onTimeOver()`経由でも同じ「城が陥落しました」のテキストが表示されてしまう

### 復活カウントダウン

**良い点:**
- 巨大な数字(fontSize=80)で視認性が高い
- 色変化(赤→オレンジ→黄)で残り時間の緊迫感がある
- 「REVIVING...」の点滅テキストで状態が明確

**問題点:**
- 特になし。実装品質が高い

## 6. 増強・属性UI

### 増強選択UI (GameState.augmentSelect)

**良い点:**
- ティア別の色分け(グレー/青/金)が明確
- 伝説ティアの金色グローリングパルスが特別感を演出
- カテゴリアイコンでの視覚的分類
- ラウンド番号の表示で「いつの選択か」が分かる

**問題点:**
- **P-21: 増強カード説明が韓国語のみ** — `aug.description`は全て韓国語(例:「메인 최대 HP +15」)。増強名は日本語(`aug.nameJp`)なのに、説明が韓国語という二重言語状態
- **P-22: タップ判定が画面横1/3分割** — 「카드를 탭해서 선택」と案内しているが、実際にはカード外をタップしても横位置に基づいて選択される。意図しないカードを選んでしまう可能性がある
- **P-23: 選択確認がない** — レベルアップバフと同様、タップ即確定。増強は1ステージ3回しかないため、バフカード以上に慎重な選択が求められる。「本当にこれを選びますか?」の確認が必要
- **P-24: 現在保有している増強の一覧が確認できない** — 増強選択画面では新しい3択だけ表示される。既に取得した増強との相乗効果(L-08 永遠の契約など)を判断する情報がない

### 属性UI

**良い点:**
- HUDのHPバー横に属性アイコンが表示される
- Row3右端にも属性アイコンがあり二重確認可能
- 属性バフカード(火/水/地/電気/闇の加護)の名前にアイコン付き

**問題点:**
- **P-25: モンスターの属性がUI上で表示されない** — 敵にどの属性が設定されているかがプレイ中に分からない。相性判断に必要な情報が欠けている(ダメージ数字の色変化のみで推測するしかない)
- **P-26: 属性相性の説明がゲーム内に存在しない** — 「火→地→電気→水→火」の四すくみ関係を知る手段がない。チュートリアルまたは属性説明画面が必要

## 7. 問題点と改善提案

### 優先度: 高 (リリースブロッカー)

| # | 問題 | 改善提案 |
|---|------|---------|
| P-04 | HUDとStageProgressの情報重複 | `_renderStageProgress()`を削除し、HUDの1段目/2段目に統合する。フォントサイズの統一(12-14px) |
| P-09 | スティックのノブが入力状態を反映しない | `_renderVirtualStick()`で`_stickKnobPos`を使用してノブ位置を描画する |
| P-08 | スティック入力が画面全体で反応する | `onDragStart`でスティック中心(80, size.y-110)から半径80px以内のタッチのみ`_stickActive=true`にする |
| P-12/P-16/P-21 | テキストの言語混在 | 全UIテキストを日本語(対外)または英語(ゲーム内表記)に統一。韓国語はコード内コメントのみ |

### 優先度: 中 (UX改善)

| # | 問題 | 改善提案 |
|---|------|---------|
| P-02 | 戦闘開始カウントダウンなし | `_startRound()`後に2秒の「3, 2, 1, GO!」カウントダウン表示を追加(スポーン開始を遅延) |
| P-14 | レベルアップ/増強選択後のフィードバックなし | カード選択時に0.3秒のスケールアニメーション+画面フラッシュを追加 |
| P-17/P-18 | ショップの購入フィードバック・確認なし | 購入時にアイテムカードの色変化(緑フラッシュ)+確認タップ(2段階) |
| P-22/P-23 | 増強タップ判定が広すぎる・確認なし | カード領域のみ判定に修正。1タップで選択ハイライト→もう1タップで確定の2段階 |
| P-19 | ゲームオーバーにゴールド表示なし | 「獲得ゴールド: XXG」を統計セクションに追加 |
| P-06/P-07 | WeaponInfoとHUD Row3が冗長 | WeaponInfoを削除(開発用デバッグ情報)。HUD Row3のゴールド表示は右下に統合し削除 |

### 優先度: 低 (品質向上)

| # | 問題 | 改善提案 |
|---|------|---------|
| P-01 | ローディングが形式的 | ロード画面にゲームTips表示、または省略してroundSelectに直接遷移 |
| P-03/P-20 | 時間切れとゲームオーバーの区別なし | `_lastStageClear`にfailure理由フラグを追加し、「時間切れ」テキストを分岐 |
| P-10 | スティックとスキルボタンの距離 | スキルボタンのY位置を-20px(上方向)に調整 |
| P-11 | スキルボタンの判定が小さい | 判定半径を44px以上に拡大(Apple HIG準拠) |
| P-13 | バフ名が英語略語のみ | サブテキストとして日本語名を追加(例: ATK UP → 攻撃力強化) |
| P-24 | 保有増強一覧の確認手段なし | 一時停止画面または増強選択画面に現在の増強リストを小さく表示 |
| P-25/P-26 | 属性情報の不足 | モンスターHPバー横に属性アイコン表示。一時停止画面に属性相性図を追加 |
| P-15 | ラウンドクリア中のスティック非表示 | roundClear中もスティックUIを表示する(半透明)、または移動入力も受け付けないようにする |

## 付録: 画面遷移フロー概念図

```
[ゲーム起動]
    │
    ▼
[loading] ──0.5秒──▶ [roundSelect] ──ラウンドタップ──▶ [playing]
                      ▲  ▲                              │ │ │ │
                      │  │                              │ │ │ │
                      │  └──────── paused ◀──────────────┘ │ │ │
                      │                                    │ │ │
                      │        ┌── levelUp ◀───────────────┘ │ │
                      │        │   (バフ3択)                   │ │
                      │        └──▶ playing                   │ │
                      │                                       │ │
                      │  ┌─ roundClear ◀──────────────────────┘ │
                      │  │    (3秒)                              │
                      │  ├──▶ augmentSelect ──▶ playing(次R)     │
                      │  ├──▶ playing(次R)                       │
                      │  └──▶ shopOpen ──▶ result(クリア)         │
                      │                                          │
                      └────────── result ◀───────────────────────┘
                               (ゲームオーバー or クリア)
```
