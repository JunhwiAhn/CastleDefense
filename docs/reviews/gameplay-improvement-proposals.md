# ゲームプレイ改善提案

> 作成日: 2026-03-11
> 担当: Planner
> 参照: 新ゲーム仕様書.md / 全specs / P-3-2〜P-3-4レビュー / castle_defense_game.dart実装
> 方針: 既存レビュー（P-3-2バランス・P-3-3 UXフロー・P-3-4バフカード体感）で**既に指摘済みの問題は除外**し、新規の改善提案のみを記載する

---

## 1. 概要

本ドキュメントは、実装済みの全システムを横断的に分析し、「ゲームプレイ体験の質」を向上させる改善提案をまとめたものである。既存レビューがバランス数値・UXフロー・バフ体感に焦点を当てていたのに対し、本提案は以下の上位テーマに集中する:

- リプレイ性と飽き対策
- 戦略的意思決定の深さ
- モンスター・ステージの多様性
- キャラクター役割の差別化と活用
- エンドコンテンツ・やりこみ要素
- チュートリアル・オンボーディング
- パーティ編成の面白さ

---

## 2. リプレイ性・飽き対策

### 2.1 現状分析

現在のゲームループは「ラウンド開始 → モンスター出現 → 撃破・XP回収 → レベルアップ → ラウンドクリア → (増強選択/ショップ) → 次ラウンド」の繰り返しである。このループ自体はVampire Survivors系として王道だが、以下の構造的問題がある:

**全ラウンドで敵の行動パターンが同一**: 通常・ミニボス・ボスの3種しかおらず、いずれも「城に向かって直線移動するだけ」。R1もR20も敵の挙動は数値（HP・速度・数）が違うだけで、プレイヤーが取るべき行動に変化がない。

**ラウンド間の「驚き」がない**: ミニボスの出現タイミング（ラウンド中盤）やボスの出現（残り5体）は固定パターンであり、2周目以降は予測可能になる。ランダムイベントやラウンド固有のギミックが存在しない。

**ビルドパスの収束**: バフカード14種 + 増強34種は数としては十分だが、P-3-4で指摘されている通り「攻撃力UP/攻撃速度UPが常に最適」の状態が続く。ビルドパスが実質1〜2本に収束しやすい。

### 2.2 提案

**A. ランダムイベントシステムの導入**

ラウンド開始時に低確率（20%程度）で以下のようなイベントをランダム発生させ、プレイに変化を加える:

| イベント名 | 効果 | 発生条件 |
|-----------|------|---------|
| モンスターラッシュ | スポーン間隔が0.5倍に。代わりにXPドロップ2倍 | R3以降 |
| 濃霧 | タワー射程が50%に低下。メインキャラの重要度が上がる | R4以降 |
| 宝の雨 | ゴールドドロップが3倍。敵HPも1.5倍 | R5以降 |
| 属性嵐 | 全モンスターの属性がランダムに変化。相性対応力が試される | R6以降 |
| 静寂の間 | スキルゲージが蓄積しない代わりに、タワー攻撃力が2倍 | R8以降 |

これにより「同じR3でも毎回違う体験」が生まれ、リプレイ性が大幅に向上する。

**B. メインキャラの自動攻撃パターンに変化を追加**

現在のメインキャラは「最寄り敵を自動攻撃」のみ。以下の攻撃モード切替を検討:

- **集中モード（デフォルト）**: 最寄り1体に集中攻撃
- **範囲モード**: 攻撃力50%だが2体同時攻撃（スティック2回タップで切替）

これにより、密集時/ボス時でプレイヤーに能動的な判断を促す。

## 3. 戦略性・意思決定

### 3.1 現状分析

プレイ中の意思決定ポイントは以下の通り:
1. **メインキャラの移動**: XPジェム回収 vs 城防衛のポジショニング
2. **レベルアップバフ選択**: 3択から1つ
3. **増強選択**: ラウンド2/4/5後に3択
4. **ショップ購入**: ステージクリア後
5. **必殺技の使用タイミング**: ゲージ100%時

問題は**戦闘中のリアルタイム判断が「移動」と「必殺技タイミング」の2つしかない**こと。タワーは完全自動であり、プレイヤーが介入できない。攻撃も自動であるため、戦闘中の操作感は「スティックを倒してXPジェムを拾いに行く」が中心になりがち。

### 3.2 提案

**A. タワー優先ターゲットの手動切替**

現在タワーは「城に最も近い敵」を自動で狙う。これに**手動でタワーのターゲットを指定する機能**を追加する:

- 画面上の敵をタップすると、全タワーがその敵を集中攻撃（5秒間 or 敵死亡まで）
- ボスやミニボスへの集中砲火の判断がプレイヤーに委ねられる
- 実装コストは低い（`_findTargetForTower`のロジックに`_manualTarget`フラグを追加するのみ）

**B. タワースロットのリアルタイム配置変更**

現在タワーは城中心±70pxの固定4箇所。ステージ中にタワー配置を変更できない。以下の仕組みを検討:

- ラウンドクリア時のインターバル3秒間に、タワースロットの入替（例: T1とT3を交換）を可能にする
- 敵のスポーン方向に応じて「上方向に火力を集中させる」等の戦術が生まれる
- UIは既存のタワースロット枠タップで実装可能

**C. 必殺技バリエーション**

現在の必殺技は「全画面999ダメージ」の1種類のみ。メインキャラのクラスに応じた必殺技バリエーションを追加すると、パーティ編成の意味が深まる:

| メインキャラ役割 | 必殺技効果 |
|---------------|---------|
| 물리딜러(物理) | 全画面999ダメージ（現行通り） |
| 마법딜러(魔法) | 全敵に5秒間の属性状態異常付与 |
| 탱커(タンカー) | 10秒間城バリア + 全モンスター3秒間バインド |
| 성직자(聖職者) | 城HP50%回復 + 全味方5秒間攻撃力50%UP |
| 유틸리티(ユーティリティ) | 全画面XP/ゴールド回収 + 10秒間スキルゲージ蓄積3倍 |

## 4. モンスター・ステージ多様性

### 4.1 現状分析

**モンスターの行動パターンが1種類**: 通常・ミニボス・ボスの全てが「城に直線移動」のみ。唯一の差異は速度とHP。タンカーのアグロに引き寄せられる機能はあるが、モンスター側の能動的な行動バリエーションがゼロ。

**ステージ間の変化が属性のみ**: Stage 1=地、Stage 2=火...と属性テーマは変わるが、モンスターの見た目と属性以外にゲームプレイの変化がない。地形・障害物・特殊ギミックが存在しない。

### 4.2 提案

**A. モンスター行動パターンの追加（3タイプ）**

現在の「直線型」に加えて以下の行動パターンを導入する:

| パターン名 | 行動 | 出現条件 |
|-----------|------|---------|
| 直線型（現行） | 城に向かって直線移動 | 全ラウンド |
| 迂回型 | 城に直線ではなくS字カーブで接近。タワーの射程を避けるように動く | R6以降、通常モンスターの20%を置換 |
| 高速突進型 | 通常の2倍速で移動するがHP半減。タワーが反応する前に城に到達する脅威 | R8以降、通常モンスターの10%を置換 |
| 分裂型 | 倒されると小モンスター2体（HP=1, 速度=通常）に分裂 | R11以降、ミニボスの代替として |

これにより「全モンスターが同じ動き」の単調さが解消され、プレイヤーは状況に応じた対応を求められる。

**B. ステージ固有ギミックの導入**

各ステージに属性テーマだけでなく、ゲームプレイに影響するギミックを追加:

| ステージ | テーマ | ギミック |
|---------|--------|---------|
| Stage 1 | 森 | なし（チュートリアル） |
| Stage 2 | 火山 | 10秒ごとにランダム位置に溶岩プール出現（3秒間、触れた味方/敵に持続ダメージ） |
| Stage 3 | 海岸 | 潮の満ち引き（30秒周期で画面下半分の移動速度が-30%。敵も味方も影響） |
| Stage 4 | 嵐の塔 | 5秒ごとにランダム敵1体に落雷（即死。ただしボスには無効） |
| Stage 5 | 暗黒城 | 視界制限（メインキャラ周囲半径150px以外が暗く、敵が見えない） |

これによりステージ攻略に「そのステージ専用の戦略」が必要になり、全ステージを同じプレイで突破する単調さが解消される。

## 5. キャラクター差別化

### 5.1 現状分析

`character_enums.dart`には5つの`RoleType`（tanker, physicalDealer, magicDealer, priest, utility）と15の`ClassType`が定義されている。しかし実際のゲームプレイでは:

**タワーユニットの個性が薄い**: タワーは「攻撃速度と射程の数値差」でしか差別化されていない。궁수（弓手）は3発同時、총잡이（ガンスリンガー）はバースト射撃、마법사（魔法師）はスプラッシュと攻撃パターンは異なるが、プレイヤーの体感としては「どのタワーも敵を自動で攻撃するだけ」に見えがち。

**メインキャラのクラスによるプレイ体験の差が小さい**: 物理딜러でも마법딜러でも、操作は「スティックで移動してXPを拾う」で同じ。攻撃は自動であるため、クラスの違いが「射程と弾のビジュアル」に留まる。

**유틸리티(ユーティリティ)ロールの存在意義が不明確**: engineer, alchemist, trickster, assistantの4クラスがutilityロールだが、戦闘中の固有メリットが不明瞭。

### 5.2 提案

**A. クラス固有パッシブスキルの導入**

各ClassTypeに戦闘中に自動発動するパッシブスキルを付与し、「このクラスをメインに据える理由」を明確にする:

| ClassType | パッシブスキル | 効果 |
|-----------|-------------|------|
| warrior | 鋼の守り | メインキャラ被ダメージ-20% |
| archer | 精密射撃 | クリティカル率15%（ダメージx2） |
| gunslinger | 弾幕展開 | 攻撃速度が敵数に比例して上昇（画面内敵10体以上で+20%） |
| pyromancer | 残り火 | 倒した敵の位置に3秒間の火柱（通過する敵にダメージ） |
| cryomancer | 氷の道 | メインキャラの移動経路に2秒間の氷跡（通過する敵の移動速度-30%） |
| druid | 大地の恵み | 10秒ごとにランダムな位置にXPジェム1個を自然生成 |
| engineer | 自動砲台 | メインキャラの現在位置に10秒間の小型タレットを設置（30秒クールダウン） |
| necromancer | 魂の回収 | 敵を倒すたびに3%の確率でゴーストを召喚（5秒間、最寄り敵を自動攻撃） |
| vampire | 血の渇き | HP50%以下で攻撃力+30%。吸血10%（攻撃ダメージの10%をHP回復） |
| trickster | 分身 | 20秒ごとに3秒間の分身を生成（メインキャラと同じ攻撃を行う幻影） |

**B. タワー配置のシナジーボーナス**

隣接するタワースロットに特定の組み合わせを配置すると、シナジーボーナスが発動する仕組み:

| 組み合わせ | シナジー効果 |
|-----------|------------|
| 물리딜러 x2 (隣接) | 両方の攻撃力+10% |
| 마법딜러 + 성직자 (隣接) | 魔法タワーのスプラッシュ範囲+20% |
| 물리딜러 + 마법딜러 (対角) | 両方のクリティカル率+10% |
| 同一属性 x2 | 属性ダメージボーナス+15% |
| 4属性全て異なる | 全タワー攻撃速度+15% |

これにより「どのキャラをどのスロットに置くか」に戦略的意味が生まれる。

## 6. エンドコンテンツ

### 6.1 現状分析

現在のゲームは「Stage 1（R1-R10）→ Stage 2（R11-R20）→ ...」と際限なくラウンドが続く構造だが、R21以降は敵HP/数が線形増加するだけで新要素の追加がない。`kStageConfigs`はStage 8まで定義されているが、Stage間の質的変化は属性テーマのみ。

P-3-2で指摘されているR21以降の「難易度頭打ち」問題に加え、**プレイヤーが長期的に目指す目標が不在**である。

### 6.2 提案

**A. チャレンジモードの導入**

通常プレイとは別に、特殊条件下でのプレイを提供する:

| チャレンジ名 | 条件 | 報酬 |
|------------|------|------|
| タワーレス | タワースロット使用不可。メインキャラのみで戦う | 限定称号 |
| スピードラン | Stage 1を最短時間でクリア | タイムランキング |
| 無強化 | バフカード・増強選択不可 | 特殊コイン |
| 属性縛り | 指定された属性のキャラのみ使用可能 | 属性別報酬 |
| 無限モード | 城HP回復なし。何ラウンドまで生存できるか | ハイスコア |

**B. 実績システム**

プレイの多様性を促すマイルストーン:

| 実績名 | 条件 | 報酬 |
|--------|------|------|
| 初心者卒業 | Stage 1クリア | ゴールド500 |
| 属性マスター | 全5属性で1回以上クリア | 限定キャラ解放 |
| ビルドマスター | 全Legendary増強を1回以上取得 | 称号 |
| 鉄壁の城主 | 城HPを一度も減らさずにStage 1クリア | 限定スキン |
| 100体撃破 | 1ラウンドで100体撃破 | ゴールド1000 |
| 全クラス制覇 | 全15ClassTypeをメインキャラとして1回以上使用 | 特殊増強解放 |

**C. メタ進行（ラン間の永続成長）**

現在はステージクリア後のショップのみが「ラン間」の成長要素。以下を追加してやりこみ深度を増す:

- **永続アップグレード**: ゲームオーバー時にも蓄積ゴールドの一部を持ち越し。永続ショップで「初期城HP+10」「初期レベル+1」等を購入可能
- **キャラクター育成**: ガチャで入手したキャラクターのレベルを戦闘外で育成。レベルが高いほどタワー配置時の基礎ステータスが向上
- **図鑑**: 出会ったモンスター・取得した増強・使用したクラスを記録。コンプリート報酬あり

## 7. チュートリアル・オンボーディング

### 7.1 現状分析

現在チュートリアルは存在しない。R1が「チュートリアルラウンド」と位置づけられているが（P-2-2, P-2-6）、実際にはプレイヤーへの説明表示は一切ない。以下の情報がゲーム開始時にプレイヤーに伝えられない:

- バーチャルスティックの操作方法
- 城を守ることがゲームの目的であること（城HP=0でゲームオーバー）
- メインキャラ死亡は一時的（5秒復活）であること
- XPジェムはメインキャラで接触して回収すること
- レベルアップバフカードの選択方法
- 属性相性の存在

### 7.2 提案

**A. コンテキスト型チュートリアル（初回プレイ時のみ）**

ゲーム進行に合わせて、必要なタイミングで吹き出しヒントを表示する:

| タイミング | ヒント内容 | 表示方法 |
|-----------|---------|---------|
| R1開始直後 | 「スティックでキャラクターを動かそう」 | スティック位置に矢印アニメーション |
| 最初の敵スポーン | 「敵が城に向かってくる! 城のHPがゼロになるとゲームオーバー」 | 城HPバーをハイライト |
| 最初のXPジェムドロップ | 「青い宝石に触れてXPを回収しよう」 | ジェムを指す矢印 |
| 最初のレベルアップ | 「バフカードを1つ選んで強化しよう」 | カードの1つを脈動ハイライト |
| R3（ミニボス初登場） | 「強敵出現! 倒すまで城にダメージを与え続ける」 | ミニボスに警告マーク |
| R5（ボス初登場） | 「ボス登場! 必殺技ゲージが溜まったらSKILLボタン!」 | SKILLボタンをハイライト |
| 初の増強選択 | 「増強はステージ中ずっと効果が続く強力な強化です」 | 増強カードを強調表示 |

**B. ヘルプ画面（一時停止メニュー内）**

一時停止メニューに「ヘルプ」ボタンを追加し、以下の情報をいつでも確認可能にする:

- 操作説明（スティック、SKILLボタン）
- ゲーム目的（城防衛）
- 属性相性表
- バフカード一覧と効果
- 取得済み増強一覧

## 8. パーティ編成の面白さ

### 8.1 現状分析

`partySlots[5]`でメイン1体+タワー4体を編成するが、現状では「強いキャラをそのまま置く」以外の戦略が薄い。属性シナジーボーナス（3属性以上で+5%ダメージ等）は仕様に存在するが、体感しにくい微量な効果に留まる。

ガチャでキャラクターを入手するシステムが存在するため、「どのキャラをどのスロットに配置するか」に深い意思決定があるべきだが、現在はRoleType（역할）によるタワー適性の差が弱い。

### 8.2 提案

**A. スロット適性ボーナス**

「メインスロットに向いているクラス」と「タワースロットに向いているクラス」を明確に差別化:

| クラスカテゴリ | メインスロット適性 | タワースロット適性 |
|-------------|----------------|----------------|
| 近接(warrior, vampire等) | S（接近戦の操作性を活かせる） | C（タワーは固定位置なので近接の射程が不利） |
| 弓/銃(archer, gunslinger) | A | S（固定位置からの安定射撃） |
| 魔法(pyromancer, cryomancer等) | A（属性状態異常を能動活用） | A（スプラッシュで城防衛） |
| 回復(priest, pastor) | B（移動しながら自己回復） | S（城HP回復のコア） |
| ユーティリティ(engineer等) | S（パッシブスキルの恩恵大） | B |

**不適性ペナルティ**: 近接クラスをタワーに配置すると攻撃範囲が極端に狭い（半径50px等）ことを視覚的に表示し、プレイヤーに「このキャラはメイン向き」と自然に学ばせる。

**B. 属性シナジーの体感向上**

現在の属性シナジー（3属性+5%、4属性+10%）は微量すぎて体感できない。以下の強化を提案:

| チーム属性数 | 現行ボーナス | 提案ボーナス |
|-----------|-----------|-----------|
| 2属性 | なし | 全キャラの状態異常発動率+5% |
| 3属性 | ダメージ+5% | ダメージ+10% + 状態異常持続時間+1秒 |
| 4属性 | ダメージ+10%, 状態異常+0.5秒 | ダメージ+15% + 状態異常+1.5秒 + 有利属性倍率x1.5→x1.7 |
| 5属性(全色) | なし(現行上限4) | ダメージ+20% + 「虹属性」（全属性有利扱い） |

5属性コンプリートを非常に強力にすることで、ガチャで多様なキャラを集めるモチベーションにもなる。

**C. 編成プリセット保存**

パーティ編成をプリセットとして保存し、ステージの属性テーマに応じてワンタップで切り替えられる機能。Stage 2（火属性）に挑む際に「水属性メイン編成」をすぐ呼び出せる。

## 9. 優先実装リスト

## 10. まとめ

本プロジェクトの現状は「Vampire Survivors x Tower Defense」のコアループが堅実に実装された段階であり、数値バランスとUIフローは既存レビュー(P-3-2〜P-3-4)で評価されている通り概ね良好である。

しかし、**リプレイ性**（毎回同じ体験の繰り返し）と**戦略深度**（戦闘中の判断が移動のみ）に構造的な課題がある。本提案のTier 1項目（チュートリアル、タワー手動ターゲット、高速突進型モンスター、クラスパッシブ、属性シナジー強化）を優先実装することで、ゲーム体験の質を大幅に引き上げることが可能である。

特に**タワー手動ターゲット指定**は最小の実装コストで最大の戦略深度向上が見込めるため、最優先で検討すべきである。
