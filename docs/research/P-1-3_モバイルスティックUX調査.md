---

## 1. 主要タイトル別スティック設計

### Archero
- **動的スポーン型**: タップした位置にスティックが出現する（固定位置ではない）
- 操作: スティックで移動 → 止まると自動射撃
- ユーザーからは「固定位置の方が良い」という声も一定数あり
- 特徴: 画面下半分のどこでも操作可能 → 親指の自然な移動範囲内で機能

### Survivor.io
- **固定位置型**: 画面左下に固定
- モバイルではスティック操作の精度に課題があるとのユーザー報告あり
- PC版（BlueStacks）では大幅に操作感が向上するという意見

### Brawl Stars
- **半動的型**: 左半分どこでもタッチ開始可能
- スティックのビジュアルはタップ位置を中心に出現
- 外径と内径（ノブ）の比率が絶妙でプレイアビリティが高い

## 2. デッドゾーン設計のベストプラクティス

### デッドゾーンとは
- スティックの中心付近での微小入力を無視する範囲
- 目的: スティックの「自然なブレ」や「誤タップ」による誤動作防止

### 推奨値
| 項目 | 推奨値 | 理由 |
|------|--------|------|
| デッドゾーン半径 | スティック外径の10〜15% | 小さすぎると誤作動、大きすぎると反応が鈍い |
| ノブ移動最大半径 | 外径の60〜70% | 親指の可動域を考慮 |
| 感度 | アナログ(移動量に比例) | デジタル8方向より360度アナログの方が自然 |

### アナログ vs デジタル操作
- **アナログ推奨**: スティックの倒し量に比例して速度が変わる → Vampire Survivors系では必須
- デジタル(8方向): シンプルだが、斜め移動が不自然

## 3. CastleDefenseへの適用推奨仕様

### スティック設計仕様 (確定推奨)

```
外径: 60px (現仕様通り)
ノブ径: 25px (現仕様通り)
配置: 画面左下固定
  → 固定位置の方がユーザーの筋肉記憶が形成されやすい
  → Survivor.io型

デッドゾーン: 外径の12% = 約7px
  → 7px未満の入力は無視
  → 意図しない微小移動を防ぐ

速度計算: (スティック倒し量 / 最大倒し量) × 最大速度(150px/s)
  → スティックを少し倒せばゆっくり、全倒しで最大速度
  → 細かい位置合わせが可能

移動方向: tan(y/x)で角度計算 → 360度全方向対応
```

### タッチ領域の推奨設定
- スティック反応範囲: スティック中心から半径80px (スティック外径60px + 余白)
- これにより、スティック外周をわずかに外れてもドラッグが継続できる

### 視覚フィードバック
- ノブはスティック外径内に収まるようにクランプ
- ノブが外径境界に当たったら最大速度 → 視覚的に「全力移動中」と分かる

## 4. 操作感のテスト項目 (Engineerへの引き渡し)

| テスト項目 | 期待値 |
|-----------|--------|
| デッドゾーン感知 | 軽く触れただけでは動かない |
| アナログ精度 | ゆっくり動かすとキャラが遅くなる |
| 方向転換の遅延 | 感知から移動まで1フレーム以内 |
| ノブのクランプ | 外径内に収まる (はみ出さない) |
| 解放時の停止 | 指を離したらすぐ停止 |
