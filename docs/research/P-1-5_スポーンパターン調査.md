## 1. Vampire Survivors — スポーンパターン設計

### 基本スポーン構造
- **ウェーブ制**: 1分ごとに1ウェーブ
- 各ウェーブで最低スポーン数と間隔を定義
- 現在数が最低数以下の場合: 補充スポーン発動
- 現在数が最低数以上でも: 各タイプから1体ずつスポーン継続

### 上限管理
- **同時画面内上限: 300体** → これ以上はスポーン停止
- ボスと特殊イベント由来のみスポーン可能（上限時も）

### マップイベント（特殊スポーン）
| イベントタイプ | 内容 |
|-------------|------|
| スウォーム | 大群が素早く画面を横断 |
| 包囲 | 高HPの敵がプレイヤーを囲む |
| ラッシュ | 一方向から大量出現 |
- イベントは**ウェーブのタイミングで固定**発動 → プレイヤーが予測可能 → 対策の面白さ

### スポーン位置
- プレイヤーの視界外から出現（画面外スポーン）
- 全方向ランダム（中心から外側一定距離）

## 2. 20 Minutes Till Dawn — スポーンパターン設計

### 基本構造
- 20分生存型ローグライト
- 時間経過で敵の種類と数が増加
- **銃器特化**: 敵の動きのパターンを読むことが重要
- 難易度が高い分、エネミーパターン学習で達成感が得られる設計

### スポーンの特徴
- 敵が「群れ」で行動する種が多い
- 時間帯によって出現する敵タイプが変化（ゲームの「リズム」を作る）

## 4. スポーンパターンのバリエーション提案

### 現在の問題点
- 全ラウンドが「全方向均等ランダムスポーン」のみ → 単調になりやすい

### 提案: 特殊スポーンパターン導入（Phase 2以降で検討）

| パターン名 | 内容 | 発動タイミング |
|-----------|------|-------------|
| **ラッシュ** | 1方向から通常の2倍速スポーン（5秒間） | 偶数ラウンドに1回 |
| **包囲** | 4方向から同時に複数体スポーン | 奇数ラウンドに1回 |
| **エリート列** | ミニボスが2〜3体連続スポーン | ラウンド中盤 |
| **ボス前ラッシュ** | ボス出現直前に通常敵を大量スポーン | ボス残り5体のタイミング |

### 推奨: まずは現仕様の全方向ランダムで実装

Vampire Survivorsも初期は全方向均等スポーンでシンプルだったため、**まずコア実装を完成させてからバリエーションを追加**する戦略が推奨される。

## 5. スポーンパターンのバランス設計指針

### 城中央配置との相性
- 敵が四方から城へ直進 → **全てのタワーが毎ラウンド活躍**できる
- Kingdom Rushのような「一方向から来る敵」ではないため、タワー配置戦略よりも「何をセットするか」が重要に

### 難易度カーブ（スポーンから考える）
| R番号 | 同時最大敵数(推定) | 難易度感 |
|-------|----------------|--------|
| R1 | 8〜10体 | チュートリアル感 |
| R2-3 | 12〜16体 | 序盤の本番 |
| R4-5 | 18〜21体 | 中盤、ボス初登場 |
| R6-10 | 24〜36体 | 後半、スポーンが激化 |
| R11+ | 40体上限到達 | 上限管理が重要 |

### 3秒インターバルの重要性
- Vampire Survivorsにはラウンドインターバルがない（連続ウェーブ）
- CastleDefenseの**3秒休憩はXPジェム回収チャンス**として機能 → Vampire Survivorsの「マグネット吸引」に相当
- この設計により「ラウンドクリア達成感」+「次の準備」の両方を提供
