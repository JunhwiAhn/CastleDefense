## 1. TFT (Teamfight Tactics) — 増強(Augment)システム

### 概要

TFTの増強(Augment)システムはSet 6「Gizmos & Gadgets」から導入された、プレイヤーのヘックスコアに永続する強化ボーナス。毎ゲームのビルド方向を大きく左右する最重要システム。

### 選択タイミング・頻度

| タイミング | ステージ (標準) | ステージ (Hyper Roll) |
|---|---|---|
| 第1回 | 2-1 | 3-1 |
| 第2回 | 3-2 | 5-2 |
| 第3回 | 4-2 | 7-2 |

- 各タイミングで**3択**から1つを選択
- 選択しない2枚は消える（持ち越しなし）
- 1試合に合計**3回**のみ増強を取得できる

### レアリティ(ティア)システム

| ティア | 色 | 強さの目安 |
|---|---|---|
| Silver | 銀色 | 弱〜中程度の補助効果 |
| Gold | 金色 | 中〜強の効果 |
| Prismatic | 虹色(プリズム) | 非常に強力、ビルドを規定するレベル |

#### 選択時のティア組み合わせ確率（例）

各選択回で提示される3枚のティア組み合わせには確率分布がある。例えばステージ2-1での一般的な構成はSilver/Silver/Silverが多く、ゲームが進むにつれGold・Prismaticの出現率が上がる。Gold/Gold/Gold提示の確率は約7%程度（パッチにより変動）。

### 増強の種類・カテゴリ

| カテゴリ | 内容 |
|---|---|
| **戦闘系(Combat)** | ダメージ・HP・シールド強化など直接戦力向上 |
| **経済系(Economy)** | ゴールド獲得・利子の増加、ロールコスト削減 |
| **アイテム系(Items)** | アイテム取得、装備強化 |
| **トレイト系(Traits)** | 特定シナジーのボーナス強化 |

#### 具体的な増強例

- **Double Trouble（Silver/Gold/Prismatic）**
  盤面に同一ユニットが2体いるとき、両方のAD・AP・防御が上昇。ユニットが3スターになると2スターのコピーを獲得。

- **Future Sight（Gold/Prismatic）**
  ゼファー(Zephyr)またはラジアントゼファーを獲得し、次の対戦相手を事前に確認できる（ユーティリティ系）。

- **The Golden Egg（Prismatic）**
  5ターン後に孵化する巨大なゴールデンエッグを獲得。ステージ4-6専用の特殊増強。

### UI・プレゼンテーション

- 選択画面ではゲームが一時停止し、3枚のカードが中央に表示される
- 各カードにはアイコン・名前・効果テキスト・ティア色が明示
- マウスオーバーで詳細効果を確認可能
- 制限時間あり（タイムアウトでランダム選択）

### バランス考慮点

- Prismatic増強はゲーム全体の方向性を規定するほど強力であるため出現率を低く設定
- 同じ増強の複数選択を防ぐ仕組みがある（同一増強は同時に提示されない）
- メタ環境によってティアリスト評価は毎パッチ変動

## 2. Vampire Survivors — レベルアップ選択・アイテム進化システム

### 概要

Vampire Survivorsでは経験値でレベルアップするたびに武器・パッシブアイテムの選択が行われる。さらに特定条件を満たすと武器の「進化(Evolution)」が発生する。

### 選択タイミング・頻度

- **レベルアップのたびに毎回**（頻度は経験値獲得速度に依存）
- ゲーム開始直後から終盤まで継続的に発生
- レベル上昇ごとに必要経験値が増加するため、後半は選択間隔が広がる傾向

### 選択の仕組み

- レベルアップ時にゲームが**一時停止**
- **3択または4択**（キャラクターや状況により変化）のカードが表示される
- 選択肢は武器とパッシブアイテムの混在
- すでに6種類の武器または6種類のパッシブを所持している場合、その枠の新規アイテムは提示されない
- 既にMAXレベルのアイテムも提示されない

### レアリティ・ティア

Vampire Survivorsには明示的なレアリティ色分けはなく、**強化回数(レベル)**によって差別化される：

| 段階 | 内容 |
|---|---|
| 新規取得 | アイテムをLv.1で取得 |
| 強化選択 | 既存アイテムのレベルを+1（Lv.2〜Lv.8） |
| 最大レベル(Lv.8) | 進化の前提条件を満たす |

### 進化(Evolution)システム

| 条件 | 内容 |
|---|---|
| 武器レベル | 対象武器をLv.8（最大）にする |
| 触媒パッシブ | 対応するパッシブアイテムを所持する |
| ボスチェスト | 10分経過後のボス撃破でドロップするチェストを開く |

- 進化時、触媒パッシブは**消費**される（後で再取得可能）
- 進化後の武器はさらに強力な上位形態に変化
- ユニオン(Union)：同種の武器2本を組み合わせてさらに強力なものにするシステムも存在

### アイテムカテゴリ

| カテゴリ | 例 |
|---|---|
| **攻撃系武器** | Magic Wand, Knife, Whip |
| **範囲・設置系武器** | Holy Water, Garlic, Santa Water |
| **召喚系武器** | King Bible, Peachone |
| **攻撃強化パッシブ** | Spinach（ダメージ+）, Bracer（速度+） |
| **生存パッシブ** | Armor, Wings（移動速度+）, Crown |
| **進化触媒パッシブ** | Hollow Heart, Spellbinder |

### UI・プレゼンテーション

- シンプルなカード3〜4枚の横並び
- アイコン・アイテム名・効果テキストが明示
- 現在のレベルが色・数字で表示
- 選択は即時反映、ゲーム再開

### バランス考慮点

- 最大所持スロット制限（武器6・パッシブ6）で無限強化を防止
- 触媒パッシブの消費で進化後の再構築コストを発生させる
- ステージ内で拾えるアイテムでスロット外の強化も可能（上限回避手段として存在）

## 3. Brotato — ウェーブ間ショップ＆アップグレードシステム

### 概要

Brotato（ブロテト）はウェーブ戦闘→ショップフェーズの繰り返し構造で、ショップでのアイテム購入と経験値によるアップグレード選択の2軸で強化が進む。

### 選択タイミング・頻度

#### ショップフェーズ（ウェーブ後毎回）

- 各ウェーブ終了後、**自動的にショップへ遷移**
- 最大**4つのアイテム**がランダム提示
- ゴールドを消費してアイテムを購入する
- リロール可能（コストはリロールごとに増加、上限なし）
- アイテムを4つ全購入すると**無料リロール**が1回付与

#### アップグレード選択（ウェーブ中レベルアップ時）

- ウェーブ中に経験値でレベルアップするたびに**4択**のアップグレードが提示
- プライマリステータスを直接増加させる選択肢

### レアリティ(ティア)システム

#### アイテム（ショップ）

| ティア | 色 | 出現開始ウェーブ |
|---|---|---|
| Common (Tier 1) | 白 | Wave 1〜 |
| Uncommon (Tier 2) | 青 | Wave 2〜 |
| Rare (Tier 3) | 紫 | Wave 4〜 |
| Legendary (Tier 4) | 赤 | Wave 8〜 |

- ウェーブ進行とともに平均ティアが上昇
- Luck(運)ステータスでより高ティアのアイテムが出やすくなる

#### アップグレード（レベルアップ）

| ティア | ステータス増加量 |
|---|---|
| Common | 少 |
| Rare | 中 |
| Epic | 大 |
| Legendary | 非常に大 |

### アップグレードの種類

| カテゴリ | 主要ステータス |
|---|---|
| **攻撃系** | Damage（ダメージ）, Attack Speed（攻撃速度）, Crit Chance/Damage |
| **射程・弾系** | Range（射程）, Projectile Speed, Bounces（バウンス） |
| **生存系** | HP, Armor, Dodge, Lifesteal |
| **移動系** | Speed（移動速度）, Dodge |
| **経済系** | Luck（運）, Harvesting（素材収集量） |
| **特殊系** | Engineering（タレット系）, Dodge Chance |

### ショップのアイテムカテゴリ

- **武器**: 銃・近接・魔法など多種
- **アイテム(パッシブ)**: ステータスを上げるが副作用（デメリット）を持つものも多い
  - 例: ある攻撃を強化する代わりに移動速度が下がる
- **消耗品**: ウェーブ中に使えるアイテム

### UI・プレゼンテーション

- ショップ画面は独立したUI（戦闘とは別画面）
- アイテムカードにティア色・価格・効果が明示
- 所持ステータスの変化プレビューがリアルタイム表示
- リロールボタン・購入・売却が直感的に配置
- キャラクター固有の出現武器制限あり

### バランス考慮点

- リロールコストの逓増でゴールドの無駄遣いを抑制
- アイテムのデメリット(負のステータス)がビルドの深みを生む
- キャラクターの特性(Trait)により有効アイテムが絞られ、無思考な最強アイテム選びを防止
- Luck ステータスがティア・リロール品質・消耗品確率のほぼ全てに影響するため「究極のステータス」と呼ばれる

## 4. Hades — 恩恵(Boon)選択システム

### 概要

Hadesでは各部屋クリア後に神々から「恩恵(Boon)」を受け取る。ランごとに組み合わせが変わり、ビルドの柔軟性と再プレイ性を支えている。

### 選択タイミング・頻度

- **各部屋クリア時**に報酬が発生（全ての部屋がBoonとは限らない）
- 部屋アイコンで事前に報酬タイプを確認可能
- Boon報酬部屋：3択から1つ選択
- Charon's Shop（カロンのショップ）でも150オボルスで購入可能
- 一度取得したBoonはラン終了（死亡・脱出）まで保持

### 恩恵の提供神と種類

| 神 | 主な強化傾向 |
|---|---|
| Zeus | 雷・連鎖攻撃 |
| Poseidon | 吹き飛ばし・ゴールド強化 |
| Athena | 反射・防御 |
| Artemis | クリティカル特化 |
| Ares | 出血・呪い |
| Aphrodite | 弱体化・魅了 |
| Demeter | 凍結・鈍足 |
| Hermes | 速度強化 |
| Chaos | リスクとリターンのギャンブル系 |

### レアリティ(ティア)システム

| ティア | 効果の強さ | 備考 |
|---|---|---|
| Common | 基本値（例: 50ダメージ） | 最も出現頻度高 |
| Rare | 基本値より約50%強 | 中程度の出現頻度 |
| Epic | さらに強力 | 低出現頻度 |
| Heroic | 最高レアリティ | 特定条件でのみ登場 |
| **Legendary** | 固有の強力効果（特別枠） | 前提Boon2つ以上必要 |
| **Duo** | 2神のBoonを組み合わせた強力効果 | 両方の神のBoonが前提 |

- Heroic Boonの入手方法:
  1. 取得時にEpicのBoonをHeroicに昇格させるオファーを受ける
  2. アスポデル地区でエウリュディケに会い「Ambrosia Delight」を選ぶ
  3. デメテルの「Rare Crop」Boonでの昇格

### レベルアップ（Pom of Power）

- 各BoonはLv.1から取得し、「Pom of Power」アイテムで強化可能
- レベル強化はレアリティに関係なく固定量上昇
- ただしHermesのBoonはPomで強化不可

### 特殊恩恵カテゴリ

| カテゴリ | 内容 |
|---|---|
| **攻撃系(Attack)** | 通常攻撃を強化 |
| **特殊攻撃系(Special)** | 武器の特殊攻撃を強化 |
| **キャスト系(Cast)** | 魔法・設置スキルを強化 |
| **ダッシュ系(Dash)** | ダッシュ行動を強化 |
| **コール系(Call)** | 神の加護(大技)を強化 |
| **パッシブ系** | 受動的ステータス強化 |

### UI・プレゼンテーション

- 3択カード形式
- 各カードに神のポートレート・Boon名・効果・レアリティが表示
- 神固有の色テーマ（Zeus=黄、Poseidon=青、Ares=赤など）で視認性確保
- 選択後すぐにダイアログ（神との会話）が流れ、ゲームリズムに組み込まれている

### バランス考慮点

- Legendary・DuoBoonは前提条件により自然に序盤〜中盤でのビルド誘導が発生
- Trial of the Godsルーム（2神が競合）での選択がトレードオフを生む
- Chaos Boonはデメリットと引き換えにリターンが大きいリスク設計
- Hermes Boonはレベル上昇不可という非対称設計でバランスを調整

## 5. Slay the Spire — カード・レリック選択システム

### 概要

Slay the SpireはローグライクデッキビルドRPGで、戦闘後のカード報酬とチェスト・ショップ・ボス後のレリック取得を組み合わせた強化システムを持つ。

### 選択タイミング・頻度

| 報酬タイプ | タイミング |
|---|---|
| カード報酬（通常） | 通常戦闘勝利後 |
| カード報酬（エリート） | エリート戦闘勝利後 |
| カード報酬（ボス） | ボス戦勝利後（レア限定） |
| レリック（チェスト） | マップ上のチェスト部屋 |
| レリック（ボス） | ボス撃破後 |
| レリック・カード（ショップ） | ゴールドで購入 |

- 通常カード報酬: **3択**から1枚を選ぶ（またはスキップ可能）
- スキップするとスパークル（ランダム報酬）が蓄積される場合あり

### レアリティシステム

#### カード

| レアリティ | 色 | 通常戦闘出現率 | エリート出現率 |
|---|---|---|---|
| Common | グレー | ~60% | ~50% |
| Uncommon | 青 | ~37% | ~40% |
| Rare | 金色 | ~3% | ~10% |

- **動的オフセットシステム**: Commonカードが出るたびにレア出現率が+1%増加し、Rareが出ると-5%にリセット（最大+40%まで蓄積）。これにより「長い間Rareが出ないことはない」ことが保証される。
- ボス後のカード報酬は**Rare限定**。

#### レリック

| レアリティ | 通常入手確率 |
|---|---|
| Common | 50% |
| Uncommon | 33% |
| Rare | 17% |

チェスト種別ごとに確率が変わる（小チェスト:Common寄り、大チェスト:Uncommon寄りなど）。

#### 特殊レリックカテゴリ

| カテゴリ | 入手経路 |
|---|---|
| Boss Relic | ボス戦のみ |
| Event Relic | 特定イベント結果のみ |
| Shop Relic | 商人から購入のみ |
| Starter Relic | ゲーム開始時のみ |

### カードの種類

| 種別 | 機能 |
|---|---|
| **Attack** | 敵にダメージ |
| **Skill** | バフ・デバフ・ドロー効果 |
| **Power** | 永続的なパッシブ強化 |
| **Curse** | 手札を汚染する負の効果（強制取得ケースあり） |

### レリックの種類（機能別）

| カテゴリ | 例 |
|---|---|
| **攻撃強化** | Akabeko（初戦闘の攻撃+8ダメージ） |
| **ドロー・マナ** | Bag of Preparation（初ターン+2ドロー） |
| **経済系** | Maw Bank（金貨+12/フロア） |
| **カード変革** | Frozen Egg（カード報酬のPowerが強化済みで出現） |
| **ボス限定** | Runic Dome（相手の意図が見えなくなる代わりに強化） |

### UI・プレゼンテーション

- カード報酬: 3枚のカードが中央に展開、詳細はホバーで確認
- レリック: 1〜3枚（ボスは3択）のアイコン＋説明文
- マップ画面で次の部屋の報酬タイプをアイコンで事前確認可能
- カードは視覚的に「色（レアリティ）」「種別（攻撃/スキル/パワー）」が一目で分かる設計

### バランス考慮点

- スキップ可能な設計でデッキの「薄さ」を維持するリワードが存在
- 動的レア出現率でRng運ゲーのフラストレーションを緩和
- ボスレリックは強力だがデメリット付きが多く（Runic Domeなど）、選択に意味を持たせる
- Curse（呪いカード）によるデッキ汚染でリスクある選択肢が生まれる

## 6. 20 Minutes Till Dawn — レベルアップアビリティ選択システム

### 概要

20 Minutes Till Dawnは20分生き残るローグライトシューターで、Vampire Survivorsに近い構造を持ちながら、武器進化ツリーとキャラクター固有アップグレードが特徴的な強化システムを持つ。

### 選択タイミング・頻度

- **レベルアップのたびに**アップグレード選択が発生
- 経験値は敵撃破で獲得（継続的に発生）
- Lv.20で武器進化の最終選択が一度発生
- エリート敵撃破時に**チェスト**がドロップ → キャラクター固有アップグレードを入手

### 選択の仕組み

- レベルアップ時にゲームが**一時停止**
- **3択**（通常）から1つを選択
- スターターキャラ「Shana」は各レベルで**1回リロール**可能
- 同じツリーの上位選択肢は前提アップグレードを取得後に提示される

### アップグレードツリー構造

- 全**25ツリー・100アップグレード**
- 各ツリーは4段階の進化:

```
[Tier 1] 1択（ツリー入口）
    ↓
[Tier 2] 2択（どちらか一方でTier 3へ進める）
    ↓
[Tier 3] 1選択（さらに強力）
    ↓
[Tier 4 / Ultimate] 最終形態
```

- Tier 2は両方取得可能（ただし1つでTier 3へ進める）
- Lv.20の武器進化は3択から1つ（以後変更不可）

### レアリティ

明示的なレアリティ色分けは存在しないが、ツリー内の**段階（Tier）**が事実上のレアリティとして機能。上位Tierほど効果が強力。キャラクター固有アップグレードは**チェストからのみ入手**できる特別枠。

### アップグレードカテゴリ

| カテゴリ | 内容 |
|---|---|
| **ダメージ系** | 通常ダメージ増加、クリティカル |
| **属性系（元素）** | Fire / Frost / Lightning ダメージ（3属性） |
| **リロード系** | リロード速度、弾数増加 |
| **攻撃速度系** | 発射レート強化 |
| **弾道系** | 発射数増加、貫通、バウンス |
| **移動系** | 移動速度強化 |
| **生存系** | HP・シールド増加 |
| **召喚系** | 味方の召喚物強化 |
| **キャラ固有** | 特定キャラのみ有効な特殊効果 |

### シナジーシステム

- 特定の2つのアップグレードを取得することで**シナジーボーナス**が発生
- シナジーが条件を満たすと、それ自体がレベルアップ選択肢として出現
- ほぼ全シナジーで2アップグレードの組み合わせが必要

### 武器進化（Lv.20）

- Lv.20到達時に武器の進化先を**3択**から選ぶ
- 一度選んだら変更不可
- Vampire Survivorsと違いアイテム等の前提条件なし（レベルだけで解放）
- ゲーム内11種類の武器それぞれに進化先が存在

### UI・プレゼンテーション

- ダークな背景に3択カード
- 各カードにアイコン・名前・効果テキスト
- ツリー内の現在位置は確認できるが常時表示ではない
- キャラクター固有アップグレードは別のチェスト演出でUIが異なる

### バランス考慮点

- ツリー構造により前提取得が必要で、無計画な選択では上位効果に届かない
- 武器進化の3択が最大のトレードオフ（元に戻せない）
- 3属性の元素ダメージで特化ビルドか分散ビルドかの選択を促す
- Shanaのリロール能力が初心者にも対応できる学習曲線を提供

## 7. 横断比較・設計示唆

### 各ゲームの主要パラメータ比較

| 項目 | TFT | Vampire Survivors | Brotato | Hades | Slay the Spire | 20MTD |
|---|---|---|---|---|---|---|
| **選択頻度** | ゲーム中3回のみ | レベルアップ毎（高頻度） | ウェーブ毎+レベルアップ | 部屋クリア毎 | 戦闘毎 | レベルアップ毎 |
| **択数** | 3択 | 3〜4択 | 4択(ショップは最大4) | 3択 | 3択 | 3択 |
| **ティア数** | 3段階 | 実質1段階（レベルで区別） | 4段階 | 4+2特別 | 3+特殊 | ツリー4段階 |
| **スキップ可否** | 不可 | 不可 | 購入しないことは可 | 不可 | 可（カードのみ） | 不可 |
| **リロール** | なし | なし | あり（有料） | なし | なし | Shanaのみ1回 |
| **強化の永続性** | ラン内永続 | ラン内永続 | ラン内永続 | ラン内永続 | ラン内永続 | ラン内永続 |
| **ゲーム停止** | 停止 | 停止 | 別フェーズ | 停止 | 停止 | 停止 |

### 設計パターンの分類

#### A. 「ビルド定義型」（少頻度・高インパクト）
**代表: TFT**
- 選択回数が少ない（3回）分、1回の選択の重みが大きい
- Prismatic増強1枚でゲームの戦略方針が決まるほどのインパクト
- 戦略的思考を強く要求

#### B. 「継続成長型」（高頻度・中インパクト）
**代表: Vampire Survivors / 20MTD**
- レベルアップ毎に選択→小さな強化の積み重ね
- 序盤のビルド方針選択 → 中盤の方向確定 → 終盤の完成という流れ
- 1回の失敗がリカバリー可能なため初心者に優しい

#### C. 「経済統合型」（ウェーブ間ショップ）
**代表: Brotato**
- 強化選択とゴールド経済が融合
- 「何を買うか」「何を売るか」「いつリロールするか」の3つの意思決定が発生
- 経営戦略的な深みが生まれる

#### D. 「ナラティブ統合型」（部屋報酬）
**代表: Hades**
- 報酬選択がストーリー演出と一体化
- 神々との会話によりゲームへの没入感が高まる
- 部屋アイコンで事前に報酬タイプがわかる → 計画的なルート選択

#### E. 「デッキ構築型」（カード取得）
**代表: Slay the Spire**
- 強化 = デッキへのカード追加（デッキが「太る」リスクも）
- スキップ可能にすることで「取らないことの価値」を生む
- レリックと組み合わせた相乗効果設計

### CastleDefenseへの応用示唆

1. **選択頻度の設計**: 高頻度型（Vampire Survivors式）は選択疲れのリスクがある。CastleDefenseのラウンド構造では「ラウンド終了後に一度」の選択（Brotato/TFT混合型）が適切と考えられる。

2. **ティア数**: 3〜4段階が標準。2段階では差別化不足、5段階以上では複雑すぎる。**Silver/Gold/Prismatic（3段階）** または **Common/Rare/Epic/Legendary（4段階）** が推奨。

3. **択数**: 3択が最も一般的で視認性も良い。4択は情報量が増えるが選択肢の幅も広がる。モバイル画面では**3択**が最適。

4. **リロールの有無**: Brotato式のゴールドコストリロールはゲームにゴールド経済と連動した深みをもたらす。ゴールドショップが存在するCastleDefenseとの親和性が高い。

5. **前提条件による誘導**: Hadesのデュオ恩恵や20MTDのツリー構造のように、前提取得が必要な上位強化を設計するとプレイヤーに目標を与えられる。

6. **一時停止UI**: 全ゲームがモバイル/PC問わず強化選択時に**ゲームを停止**している。モバイルゲームでは特に必須。

7. **スキップ設計**: Slay the Spireのみスキップ可能。タワーディフェンス文脈では「常に強化を取りたい」心理に反するため、スキップは不要と思われる。
