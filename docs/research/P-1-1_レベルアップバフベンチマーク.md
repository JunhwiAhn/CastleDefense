# P-1-1: レベルアップバフ設計のベンチマーク調査

> GitHub Issue #1 対応
> 担当: Planner
> 参考タイトル: Vampire Survivors, Brotato, Halls of Torment

## 1. Vampire Survivors — レベルアップシステム

### XP必要量テーブル
| レベル帯 | 1レベルあたりの増加量 |
|---------|------------------|
| Lv 1→2 | 5 XP (初回) |
| Lv 2→20 | 前レベル+10 XP ずつ増加 (15, 25, 35...) |
| Lv 21→40 | 前レベル+13 XP ずつ増加 |
| Lv 41以降 | 前レベル+16 XP ずつ増加 |
| **特殊** | Lv20, Lv40でボーナス必要XP +600/+2400 (その間Growth+100%) |

### 選択肢の構成
- レベルアップ時: **3〜4択** をランダム提示（一時停止）
- 所持済みアイテムが優先的に出現（2回ロール）
- 同じアイテムは1度に1つのみ
- インベントリ満杯時: 「ゴールド獲得」「HP回復チキン」に切り替わり

### バフカテゴリ
1. **武器** (積極的な攻撃手段)
2. **パッシブアイテム** (ステータスボーナス、最大6種類まで)
3. **武器進化** (武器MAX + 対応パッシブで胸箱から発動)

### 設計の特徴
- 選択肢が常に「意味のある選択」になるよう調整
- 所持アイテムが優先→「育てている武器をより強化できる」感覚
- インベントリ上限により選択肢が収束し、後半は進化を狙う戦略性が生まれる

## 2. Brotato — アイテム選択システム

### 基本構成
- 各ウェーブ終了後にショップ画面（ゲーム停止型ではなく専用画面）
- 16の強化スロット、44キャラクター、55武器、177アイテムの組み合わせ

### 重み付けシステム
- 同じ武器を多く買うほど、そのアイテムが出やすくなる（ビルド完成を促進）
- 同クラスの武器が出る確率はターン5で最大化
- ウェーブ進行でレアリティが上昇（高ウェーブ=青・紫アイテム主体）

### コミュニティが指摘するバランス問題
- 高ウェーブになるとコモンアイテムが選択肢に出なくなる
- コモンアイテムも強力かつ安価なため、多様性が失われるという意見あり
- 混合武器ビルドは同タイプ武器ビルドよりも確率的に不利

## 3. Halls of Torment — レベルアップシステム

### 特徴
- 生存時間: **30分1ラウンド制**
- 敵を倒してジェム収集 → レベルアップ → **ランダム能力解放**
- **階層別トレイト**: トレイトにはTier(3, 6, 10)があり、そのTierを逃すと二度と取得不可
- 約1000種類のトレイトで毎ランの体験が変わる
- ランダムルートとの組み合わせで戦略的深さが増す

## 4. CastleDefenseへの応用提案

### 現仕様との比較

| 項目 | 現仕様 | Vampire Survivors | Brotato | 提案 |
|------|--------|-------------------|---------|------|
| 選択肢数 | 3択 | 3〜4択 | 4択(ショップ) | **3択で問題なし** |
| 一時停止 | あり | あり | あり(専用画面) | **あり（維持）** |
| 重複上限 | 最大5回 | アイテムごとに異なる | 無制限(重み付け) | **カテゴリ別上限（維持）** |
| XP要求増加 | Lv×5+10 | 初期は急増、後半は緩やか | — | **現仕様でバランス可** |

### 推奨調整事項
1. **XPカーブ確認**: 現仕様 `10 + (Lv × 5)` はVS初期より緩め。Lv1=10, Lv2=15は適切
2. **バフの偏り防止**: 「攻撃力UP」「攻撃速度UP」が連続して出ないよう、直前選択と同じバフの出現確率を下げる
3. **城バリア・城修復は無制限重複が正しい**: Vampire Survivorsの「チキン(HP回復)」と同じ役割で常に有用
4. **選択肢の多様性**: 3択のうち1択は「全体に効くバフ(タワー強化等)」が入るよう調整推奨
